ProAgentStore MCP

Use ProAgentStore from Codex, Claude Code, Cursor, and other MCP clients through the official MCP server. Agents should use MCP workflows instead of scraping the UI or calling private APIs directly.

Use It Now

Codex

codex mcp add proagentstore --url https://mcp.proagentstore.online/mcp
codex mcp list
# If the server shows "Not logged in":
codex mcp login proagentstore

Claude Code

claude mcp add --transport http proagentstore https://mcp.proagentstore.online/mcp
claude mcp list

Generic MCP Client

npx mcp-remote https://mcp.proagentstore.online/mcp

Local npx Proxy

npx @proagentstore/cli mcp

Authentication uses browser sign-in/OAuth where the client supports it. Existing PAGS session tokens are accepted by individual tools as a fallback.

Project Config

Add this to repos that should use ProAgentStore only through MCP:

{
  "mcpServers": {
    "proagentstore": {
      "type": "streamable-http",
      "url": "https://mcp.proagentstore.online/mcp"
    }
  }
}

For Codex project-local config in trusted repos:

[mcp_servers.proagentstore]
url = "https://mcp.proagentstore.online/mcp"
tool_timeout_sec = 120
default_tools_approval_mode = "prompt"

For stdio-only MCP clients, use the published CLI as the local proxy:

{
  "mcpServers": {
    "proagentstore": {
      "command": "npx",
      "args": ["-y", "@proagentstore/cli", "mcp"]
    }
  }
}

Agent Instructions

Use ProAgentStore only through the configured MCP server.

Do not use the web UI.
Do not call REST/GraphQL/private APIs directly.
First inspect available MCP tools/resources.
Prefer read-only tools unless the task explicitly requires changes.
Confirm before destructive actions.

Auth Scopes And Safety

OAuth connections can request read, write, runtime, and destructive scopes. If a client only requests standard OAuth/OIDC scopes such as openid email profile, ProAgentStore grants the existing default scope set for compatibility. Write, runtime, and destructive tools enforce ProAgentStore scopes server-side. Set MCP_READ_ONLY=1 on the MCP worker to force read-only mode.

ControlBehavior
dry_run: trueAvailable on mutating tools where useful; returns the planned action without changing state.
confirmRequired for destructive or overwrite-style tools: write_agent_file, batch_write_agent_files, unregister_instance_runtime, cancel_instance_task, delete_instance_knowledge, and cancel_instance.
mcp_audit_logReads recent MCP write, runtime, dry-run, denied, and destructive tool audit events for the authenticated account.

Security Model

MCP servers expose tools to agents. Keep permissions narrow, use specific tools, and avoid generic execution surfaces.
  • OAuth/browser sign-in is the default auth path.
  • OAuth scopes are enforced server-side for write, runtime, and destructive tools.
  • Tools are purpose-specific; there is no generic shell or arbitrary API proxy tool.
  • Mutating tools support dry-run previews where useful.
  • Destructive and repository overwrite tools require explicit confirmation.
  • MCP audit events are stored for authenticated OAuth sessions.
  • Browser actions are task-based and can require explicit approval.
  • Private instance runtime uses caller-owned AI credentials and refuses platform-owned AI billing.
  • Use `default_tools_approval_mode = "prompt"` in Codex for write-heavy work.
