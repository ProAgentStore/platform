Setup Guide

Connectors and Google Docs

Google Docs are imported through the Google Drive connector. A user connects their Google account once, then grants individual agent instances access to specific Drive folders.

Access model

Folder grants

  • OAuth connections are account-level and stored in user_api_keys with encrypted refresh tokens.
  • Agent access is instance-level through instance_connector_grants.
  • A folder grant authorizes that folder and descendants.
  • Imported Google Docs are copied into the selected agent instance's knowledge base.
The agent does not automatically get the whole Google account. It can only browse/import through folders granted to that instance.

Admin setup

  1. Create or open a Google Cloud OAuth Web application.
  2. Add the redirect URI https://api.proagentstore.online/v1/drive/google/callback.
  3. Enable the Drive API and request https://www.googleapis.com/auth/drive.readonly.
  4. Set Worker configuration for GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, and KEY_ENCRYPTION_KEY.
cd workers/api
pnpm wrangler secret put GOOGLE_CLIENT_SECRET
pnpm wrangler secret put KEY_ENCRYPTION_KEY

If the same Google OAuth app is used for login or Gmail, keep those callback URLs too: /v1/auth/google/callback and /v1/email/google/callback.

Connect Google Docs

  1. Open the ProAgentStore Console.
  2. Open an agent instance, then go to Settings.
  3. Click Connect Google Drive and approve access.
  4. Paste a Google Drive folder URL or folder ID into Grant folder.
  5. Go to Knowledge, click + Drive, list/search files, then import supported Google Docs, Sheets, Slides, or text files.

What clients do

Each client connects their own Google account. For shared drives or team folders, grant the agent a folder that the connected Google account can read. Existing imported documents remain in the agent knowledge base even if Drive is later disconnected.

Troubleshooting

  • Not configured: confirm GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET are set on the API worker.
  • Missing refresh token: revoke the app at Google Account permissions and reconnect with consent.
  • Cannot import a file: verify the file is inside the granted folder and the connected account can read it.
  • Unsupported file type: import Google Docs/Sheets/Slides or text-like files. Binary formats need the regular file upload path.
For production setup, keep the OAuth consent screen published and make sure the callback domain exactly matches api.proagentstore.online.
