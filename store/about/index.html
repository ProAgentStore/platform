About ProAgentStore

ProAgentStore is a marketplace for server-powered AI agents. Creators build agent templates — identity, knowledge base, guardrails, tools. Clients subscribe and run them on their own documents and data. Every agent runs on Cloudflare's edge: Workers AI for inference, Durable Objects for state, D1 for databases, R2 for storage.

How it works

Creators build

Set identity, personality, guardrails. Add knowledge docs. Choose a model. Publish to the store.

Clients subscribe

Get your own isolated instance. Add your documents. Chat or call the API. Your data stays private.

Agents run

On Cloudflare's edge. Workers AI, Durable Objects, cron. No infrastructure to manage.

Three types of agents

Agents

Full AI with conversation, memory, knowledge base, 10 tools, Workers AI backbone. The flagship type.

Workers

Always-on Durable Objects for scheduled tasks. Daily digests, monitoring, batch processing.

Tools

Stateless API endpoints. Transform, generate, analyze. No state, no conversation — pure functions.

What each agent gets

Workers AI

Llama 3.3 70B, Llama 4 Scout, Mistral, Qwen. Any model from Cloudflare's catalog.

Knowledge Base

Upload docs, paste text, import URLs. Agent answers from your documents.

Guardrails

Topic restrictions, blocked terms, response style, citations. Keep the agent on track.

10 Tools

Memory, tasks, fetch URLs, store/read files. Agents act, not just chat.

Conversation

Persistent chat history. Memory across sessions. WebSocket real-time.

API + Widget

REST API, SDK, embeddable chat widget. Use agents anywhere.

Data isolation

Creators and clients never see each other's data. Each subscription creates an isolated Durable Object with its own knowledge base, conversations, and memory. The creator defines the template; the client owns their instance.

Developer tools

CLI — pags init, pags check, pags publish. Scaffold, validate, deploy in minutes.

MCP Server — manage agents from Claude Code, Cursor, or VS Code. 10 tools: create, chat, add knowledge, analytics.

Webhook ingestion — Zapier, Make, n8n can POST documents directly into an instance's knowledge base.

API key vault — store your OpenAI, Anthropic, Google AI keys once. Encrypted AES-256-GCM. Agents access via proxy.
