Get Started

Three ways to use ProAgentStore: try an agent, subscribe to one, or build your own.

Try an agent

1

Browse the store

Visit proagentstore.online and pick an agent. Each one has a live trial — no sign-up needed.

2

Chat in the detail page

Every agent detail page has a "Try it now" chat box. 20 free messages per session.

Subscribe to an agent

1

Sign in with GitHub

Go to the Console and sign in. One click — no separate account needed.

2

Get the agent

Click "Get Agent" on any store listing. You get your own isolated instance of the agent.

3

Add your documents

Upload your files, paste text, or import URLs. The agent answers based on YOUR data. Private to you — the creator never sees it.

4

Chat or call the API

Use the console chat, or integrate via the REST API and SDK.

import { initPro } from '@proagentstore/sdk'

const agent = initPro({
  agentId: 'your-instance-id',
  token: 'your-token',
})

const { response } = await agent.chat('Summarize my Q3 report')
console.log(response)

Run the job-application agent

Give it a job URL and it drives a real browser to fill and submit the application for you.

1

Subscribe + upload your résumé

Get the job-application agent from the store, then upload your résumé into its Knowledge base.

2

Fill your Profile

In Console → Profile, add your candidate details and Job Preferences (target roles, locations, work type). The agent uses these to answer application questions — it never invents values.

3

Give it a job URL

Paste a job posting URL. The agent opens the ATS, fills the form, and submits — retrying and learning per-ATS tips. One card per job on the work board tracks each attempt. When it hits a captcha or needs a value it doesn't have, it hands off to you for a takeover, then resumes. It records the employer's confirmation email as an "Open in Gmail" link.

Code with the Coder agent

An AI coding agent that runs a coding CLI on your own machine and drives any GitHub repo.

1

Start the runner

Install the CLI and start the runner. One runner serves all your agents on that machine over a WebSocket relay — no tunnel binary, no config. You can run pags up on more than one machine; Coder pins each repo session to the runner node that owns it.

npm i -g @proagentstore/cli
pags up
2

Add a repo

In the Coder's Coding tab, add a repo — a local path (~/dev/my-repo), owner/repo, or a clone URL.

3

Start a session

Pick an engine — Claude Code, Codex, or Grok — and start a session. Watch the live terminal, let the Co-pilot summarize, or hand it an objective and let the Loop drive autonomously. Different repos can run on different connected machines.

Build your own agent

1

Install the CLI

Scaffold a complete agent project with one command.

npx @proagentstore/cli init my-agent --template worker

Three templates:

  • worker — Durable Object with conversation, memory, knowledge base, core tools
  • cron — Scheduled worker (daily digests, monitoring, batch processing)
  • api — Stateless endpoint (transform, generate, analyze)
2

Customize

Edit src/index.ts — set the system prompt, add knowledge docs, wire up tools. Or use the Console UI to configure without code.

3

Publish

Run compliance checks and publish to the store in one command.

npx @proagentstore/cli check    # 11-point compliance check
npx @proagentstore/cli publish  # creates repo, deploys, registers

Your agent appears at proagentstore.online/agents/my-agent/ with a live trial, subscribe button, and embed code.

Embed on your site

Add an agent as a chat widget on any website:

<script src="https://proagentstore.online/widget.js"
  data-agent="your-agent-slug"
  data-theme="dark"></script>

Floating chat button, expandable panel, dark/light theme. Works on any origin.

Connect via webhook

Send documents from Zapier, Make, n8n, or any service:

curl -X POST https://api.proagentstore.online/v1/public/webhook/INSTANCE_ID/ingest \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title":"Meeting Notes","content":"...","source":"zapier"}'

MCP integration

Manage agents from Claude Code, Cursor, or VS Code:

npx mcp-remote https://mcp.proagentstore.online/mcp

Use chat_with_agent only for public trial chats. For the real user runtime, call subscribe_agent, then my_instances, then chat_with_instance. Instance tools keep user knowledge and chat state separate from the creator's template agent.

list_agents -> subscribe_agent -> my_instances -> add_instance_knowledge -> chat_with_instance -> instance_messages

If the instance says it needs a Cloudflare Workers AI account ID and API token, the runtime is working correctly and waiting for caller-owned AI credentials.

Install the agent skill

ProAgentStore ships the same MCP operating workflow for Codex and Claude Code through plugin marketplaces.

# Codex
codex plugin marketplace add ProAgentStore/platform

# Claude Code
/plugin marketplace add ProAgentStore/platform
/plugin install proagentstore@proagentstore
/reload-plugins

The installed plugin bundles the proagentstore-mcp-operator skill and the ProAgentStore MCP endpoint. Users still approve MCP browser sign-in before any account action runs.

Full skill docs are at /skills/proagentstore-mcp-operator/. Agent-readable discovery files are available at /llms.txt and /skills.json.

Store your API keys

Use your own OpenAI, Anthropic, or Google AI keys for higher-quality responses. Keys are encrypted with AES-256-GCM and injected server-side via proxy — never exposed to agents.

Go to Console → Profile → API Keys.

Supported: OpenAI, Anthropic, Google AI (Gemini), OpenRouter, Groq, Together AI.

Browser-side agents?

If your agent runs in the browser (no server needed), use FreeAgentStore instead — free forever, fully private, offline-capable. ProAgentStore is for agents that need server power: cron, databases, large models, webhooks, multi-user state.
