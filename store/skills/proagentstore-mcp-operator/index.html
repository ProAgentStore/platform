proagentstore-mcp-operator

Operate ProAgentStore account state only through the deployed MCP server. Use it to create agents, subscribe to agents, run private instances, add knowledge, inspect messages, and check deployment state from Codex or Claude Code.

Install Machine-readable metadata MCP endpoint

Install

Codex

codex plugin marketplace add ProAgentStore/platform

Claude Code

/plugin marketplace add ProAgentStore/platform
/plugin install proagentstore@proagentstore
/reload-plugins

What it can do

Create agents

Use MCP creator tools to scaffold, update, deploy, and inspect ProAgentStore agents in your account.

Run private instances

Subscribe to a template agent, add your knowledge, chat with the private instance, and read its messages.

Preserve billing boundaries

The runtime waits for caller-owned AI credentials instead of silently spending platform AI tokens.

Correct runtime flow

Use public trial chat only for demos and marketplace discovery:

list_agents -> chat_with_agent

Use private instance chat for durable user work:

list_agents -> subscribe_agent -> my_instances -> add_instance_knowledge -> chat_with_instance -> instance_messages

Use FAGS runtime tools for browser-capable private instances:

subscribe_agent -> register_instance_runtime -> instance_runtime_status -> run_instance_task -> approve_instance_task -> instance_task_events

PAGS is the MCP/account control plane. FAGS is the browser runtime for Playwright, local file uploads, screenshots, and approval-gated browser actions.

Required MCP tools

  • create_agent, scaffold_agent, update_agent, my_agents
  • subscribe_agent, my_instances, chat_with_instance, instance_messages
  • add_instance_knowledge, list_instance_knowledge, delete_instance_knowledge, cancel_instance
  • register_instance_runtime, instance_runtime_status, run_instance_task, approve_instance_task, instance_task_events
  • list_agents, agent_info, platform_guide, sdk_reference

Security and account access

The skill is instructional. It does not grant account access by itself. Account actions require the user to approve ProAgentStore MCP OAuth/browser sign-in in their own MCP client.

Do not use direct database writes, private API shortcuts, browser-only workarounds, or local secrets for live account operations. The deployed MCP server is the account-operation boundary.
